# 做法 / 项目特征匹配说明

> **铁律**：项目特征每一条 = 一道施工工艺。详见 `docs/项目特征与施工工艺.md`。

## 为什么必须拆做法

同名清单（如「内墙乳胶漆」「地砖地面」）单价可以差很多，差别在**特征描述里的做法**——**逐条**核对，不是扫一眼名称：

| 对比 | 识别方式 | 库内分条 |
|------|----------|----------|
| 乳胶漆三遍 vs 两遍 | `paint_coats:3` / `2` + 工艺条目 | 不同 `method_signature` |
| 水泥砂浆贴砖 vs 瓷砖胶 | `tile_bind:mortar` / `adhesive` | 不同标准项 |
| 特征少写「防水」 | 工艺条目条数/内容不一致 | 匹配降级，不自动套价 |

## 匹配权重（默认）

- 做法标签 + **逐条工艺** **45%**
- 特征全文 **40%**
- 项目名称 **15%**

## 冲突处理

关键标签不一致，或工艺条目对不上时：

- 一致度封顶，等级降为 C
- **不自动填价**，备注写明「做法冲突」或「工艺条目不一致」

## 入库优先级

1. **综合单价分析表**（块式人材机拆解）→ **最高优先级**，含主材/人工/机械明细  
2. **清单工程量表**（含综合单价列）→ 补充项目特征与工程量  
3. 同一文件两者并存时：以分析表成本为准，清单表仅保留特征不重复入库单价  

解析模块：`src/ingest/unit_price_analysis.py`

## 扩展规则

编辑 `config/feature_rules.json` 增加 `pattern` → `key` / `value` / `label`。

入库或回填：

```bash
python app.py learn "新项目.xlsx"
python app.py backfill   # 老库补标签与工艺指纹
```
