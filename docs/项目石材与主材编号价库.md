# 项目石材与主材编号价库

## 实务口径（必读）

**ST-01、ST-02… 只是某个项目物料书里的编号，不是全国统一的石材品种。**

| 项目 | ST-01 可能是什么 | 表价 |
|------|------------------|------|
| 海德售楼处 | 某款 18mm 大理石 | 500 元/㎡ |
| 另一项目 | 完全不同的石材 | 需重新录入 |

因此：

- **不能**用 A 项目的 ST-01 表价去套 B 项目的 ST-01。
- **可以**在同一项目内，用物料书表价 + 特征工序（湿贴/拼花/波打等）拆主材与铺工。
- **要慢慢收集**的是：`项目 ref` + `编号` + **石材名称（物料书描述）** + **不含税主材价**。

## 系统里价库存在哪

| 存储 | 说明 |
|------|------|
| `project_material_prices` | 项目主材表（`project_ref` + `material_code` + `material_name` + 单价） |
| `config/project_material_supplements.json` | 物料书 xlsx 缺价时的补充（如 DISPIMG 无价） |
| `config/project_pairs.json` → `project_materials_ref` | 招标组价时绑定哪一份价库 |

组价时若配置了 `project_materials_ref`：

1. **只查本项目价库**，不跨项目混用其它 `project_ref` 的 ST 价。
2. **未录入的 ST 编号** → 不走表价工序系数，改走**整项历史**或备注「请补物料书」。
3. 工序系数（如地面 20mm ×1.22、拼花 ×1.66）**仅在本项目价库命中时**启用。

## 推荐收集流程

### 1. 有物料书 Excel（首选）

```bash
python app.py import-project-materials "清单数据资料/AI学习清单/售楼处主材料.xlsx" \
  --project "珠海海德公馆售楼处会所" --ref haide_sales_office --city 珠海
```

表内 **编号 + 名称 + 单价** 一并入库；名称即该编号在本项目对应的石材描述。

### 2. 校正后从金标准补采（慢慢积累）

```bash
python app.py calibrate --project haide_sales --learn
python app.py calibrate --project haide_basement --learn
```

`--learn` 会额外执行 **编号价采集**（写入 `project_material_prices`）：

| 项目 | 采集范围 | 说明 |
|------|----------|------|
| 售楼处 | **ST** | 拼花/波打行还原为表价（÷1.66）；不覆盖物料书 xlsx |
| 地库 | **ST、MT** | **不采 PT**（金标准主材已是工序拆价后小数，不能当表价） |

地库无物料书时，靠 learn 逐步积累 **MT-02=450**、**ST-04=450** 等；勿把清单行合价主材直接当表价入库。

### 3. 缺价补充

编辑 `config/project_material_supplements.json` 中对应 `project_ref` 或 `city:珠海` 节点。

### 4. 绑定到招标项目

`config/project_pairs.json` 中为项目增加：

```json
"project_materials_ref": "haide_sales_office"
```

然后 `python app.py deliver --project haide_sales`。

## 与其它编号

**PT / CT / WD / GL** 等同理：编号是项目级别名，优先本项目价库，再整项历史，最后市场参考价。

## 相关命令

```bash
python app.py import-project-materials "物料书.xlsx" --ref 项目ref --city 珠海
python app.py calibrate --project haide_sales --learn
python app.py deliver --project haide_sales
```
