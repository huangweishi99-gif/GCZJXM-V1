# 清单、定额与表格组价原理（本项目口径）

> 结合 GB 50500 清单计价、广联达消耗量定额、深海/理想等项目实务整理。

---

## 一、清单 vs 定额：各是什么

| 概念 | 是什么 | 谁编制 | 计量单位 | 在本系统里 |
|------|--------|--------|----------|------------|
| **工程量清单（清单）** | 招标要求的「做什么、做多少」 | 招标人 | **清单单位**（㎡、m、樘、项） | 项目名称 + **项目特征** + 单位 + 工程量 |
| **定额（消耗量标准）** | 完成某道工序的「人工+材料+机械」社会平均消耗 | 造价站/广联达 | **定额单位**（常为扩大单位） | 对应 **项目特征里的施工工艺** |

**关键**：清单是**综合实体**（一块地砖面层一项）；定额是**施工工序**（找平、铺贴、勾缝可能多条定额）。

---

## 二、清单单价怎么由定额组成

典型组价路径（装饰工程）：

```text
招标清单项（1 项，单位 ㎡，工程量 100）
    ↓ 项目特征描述 = 要做的全部工艺（相当于定额组合说明）
    ↓ 套用多条定额子目（每条有自己的定额单位）
    ↓ 换算、汇总人工费、材料费、机械费
    ↓ 加上管理费、利润 → 综合单价（元/㎡清单单位）
```

**本项目表格报价**：

- **项目名称** → 清单「做什么」（面层是什么）
- **项目特征** → **施工工艺**（应能对应到要套的定额及做法）
- **成本单价分析区** → 主材、辅材、人工、机械（由定额拆解或历史项目拆解而来）

## 三、定额单位 ≠ 清单单位（必须换算）

| 类型 | 示例 | 注意 |
|------|------|------|
| 清单单位 | 块料楼地面 **㎡** | 按饰面面积计量 |
| 定额单位 | 水泥砂浆找平 **100㎡**、铺砖 **100㎡**、轻钢龙骨 **100m** | 消耗量表头是「每 100㎡」 |
| 换算 | 清单工程量 × (定额基价对应的单位工程量) | 子目工程量要按定额单位换算后相加 |

**错误做法**：把定额子目单价（元/100㎡）直接当成清单综合单价（元/㎡）而不除以 100。

**系统要求**：入库历史价时保留**清单单位**；若从定额软件导出，需先换算到清单单位再 learn。

## 四、项目特征 = 施工工艺 = 定额做法

国标：项目特征是「本质特征」；您项目实务：

> **特征每一条 ≈ 一道工序或一道做法要求**

与定额关系：

- 定额册中每条子目有 **工作内容**（应完成哪些工序）
- 清单 **项目特征** 应覆盖这些工序的关键参数（厚度、材料、粘结方式、遍数等）
- **特征与定额做法不一致** → 不能套该定额，需换子目或调消耗量

示例（地砖）：

```text
清单名称：块料楼地面-600*600
项目特征：
  1. 10厚地砖，专用勾缝剂勾缝      ← 面层主材 + 勾缝
  2. 10mm厚水泥砂浆粘结层         ← 对应铺贴定额
  3. 30mm厚1:3水泥砂浆层          ← 对应找平定额
```

对应广联达装饰定额中可能套：找平 + 铺贴 +（勾缝已含在铺贴子目或单独子目）。

## 六、无机涂料与乳胶漆（定额口径）

清单常写「无机涂料」，定额册多列 **乳胶漆** 子目；组价时**视为同一类涂料面层**：

- 套定额：无机涂料项 → 套墙面/天棚 **乳胶漆** 相应子目（厚度、遍数看特征）。  
- 查历史价：优先匹配历史 **乳胶漆** 项的人材机。  
- 系统：`paint_name_match_score`、材料价库 `paint` 品类已做等价处理。

## 七、主材 / 辅材判定（项目目录）

来源：`主材辅材判定.xlsx` → 表 `material_catalog`

- **主材**：完工可见面层或合同单独计价材料（涂料、玻璃、不锈钢、地砖、地板…）
- **辅材**：基层、辅耗、通用材料（石膏板、腻子、龙骨、水泥、防水…）

与定额关系：定额中的「未计价材料」多为**主材**；定额价内材料多为**辅材**——与本目录结合使用。

导入命令：

```powershell
python app.py import-catalog "清单数据资料/AI学习清单/主材辅材判定.xlsx"
```

## 八、组价优先级（系统逻辑）

1. 同城同档 + **名称+特征相似** → 整项历史价（主材/辅材/人工/机械）  
2. 不够相似 → **材料规格价库**（如 600×600 地砖主材）  
3. 材料目录 → 校正某材料应归主材列还是辅材列  
4. （规划）特征匹配 **定额子目** → 按消耗量算人工辅材机械
