## 二、投标报价清单（九亿通航）

### 2.1 工作簿结构

| Sheet | 行数(约) | 是否导入组价库 |
|-------|----------|----------------|
| 封面 | 15 | 否（提取元数据：工程名、投标人、总价、编制日期） |
| 报价说明 | 3 | 否（可存为项目备注/编制范围） |
| 报价汇总 | 11 | 否（校验用） |
| **装修工程清单** | 408×23 | **是（主清单）** |
| **消防工程** | 55×8 | **是** |
| 材料设备参考品牌 | 小表 | 否（二期：材料编码 GL-01 等关联） |

> 说明：装修 Sheet 内从约第 360 行起含 **「二、安装工程」** 分部，装饰+安装在同一表，导入时需按分部路径解析，不能只看 Sheet 名。

### 2.2 元数据（封面 / 表头）

| 字段 | 示例值 |
|------|--------|
| 工程名称 | 广东省深圳市龙华区龙华数字智造中心…11层 01号/装修施工工程 |
| 投标人 | 深圳市广湾建设科技有限公司 |
| 投标总价（小写） | 2073870.306132 |
| 编制时间 | 2026 年 3 月 13 日 |
| 造价阶段 | 投标 |
| 单价口径 | **综合单价（不含税）**；汇总表含管理费 3%、税金 9% |

### 2.3 「装修工程清单」列映射（表头在第 **3** 行，索引从 0 起为第 2 行）

| 列号 | 表头名称 | 系统字段 | 用途 |
|------|----------|----------|------|
| 0 | 序号 | `seq` | 展示；**非**国标编码 |
| 1 | 项目名称 | `name` | **匹配核心** |
| 2 | 项目特征 | `feature` | **匹配核心** |
| 3 | 单位 | `unit` | **匹配硬约束** |
| 4 | 工程量 | `quantity` | 合价校验 |
| 5 | 综合单价（不含税） | `unit_price` | **历史综合单价（报价侧）** |
| 6 | 合价 | `amount` | 校验：≈ 工程量×单价 |
| 7 | 备注 | `remark` | 可选 |
| 8 | 综合单价（不含税） | — | 与 col5 **重复**，导入时忽略 col8 |
| 9～13 | 主材费、主材损耗率、人工费、辅材费、机械费 | `quote_material`, `quote_labor`, … | **报价侧人材机构成** |
| 14 | 合计 | — | 可与 col6 对照 |
| 15 | 成本单价 | `cost_unit_price` | 成本分析、利润测算 |
| 16～20 | 主材费、损耗率、人工、辅材、机械 | `cost_material`, `cost_labor`, … | **成本侧人材机构成** |

**有效数据行判定（建议规则）**

- `项目名称` 非空  
- `单位` 非空（㎡、m、个、项、套等）  
- 排除：仅分部标题行（如「装饰工程」「（一）隔墙工程」「地面工程」等，无单位）

本表估算：**约 314 行** 可入库（装修 Sheet）；消防 Sheet **约 41 行**。

### 2.4 「消防工程」列映射（表头第 3 行）

| 列号 | 表头 | 系统字段 |
|------|------|----------|
| 0 | 序号 | `seq` |
| 1 | 项目名称 | `name` |
| 2 | 项目特征 | `feature` |
| 3 | 单位 | `unit` |
| 4 | 工程量 | `quantity` |
| 5 | 综合单价（不含税） | `unit_price` |
| 6 | 合计 | `amount` |
| 7 | 备注 | `remark` |

消防表 **无人材机明细列**，成本分析仅能依赖综合单价或其它项目补录。

### 2.5 清单层级（分部路径）

表中存在三级结构，导入时应生成 `section_path` 便于检索与导出：

```
一、装饰工程
  （一）隔墙工程
    1  轻钢龙骨隔墙（100mm）  ← 数据行
  （二）财务室、财务经理室
      地面工程              ← 子分部标题（无单位）
    1  浅色水磨石砖          ← 数据行
二、安装工程
  …
```

**规则**：自顶向下维护当前路径栈；遇到无单位行则入栈为分部名；遇到有单位且有单价行则挂接当前路径。

### 2.6 与国标清单差异（重要）

| 项目 | 九亿投标表 | 深海成本分析表 |
|------|------------|----------------|
| 项目编码 | **无** 12 位国标编码 | **有**（如 `010802004001`） |
| 匹配主键 | 名称 + 特征 + 单位 | 编码 + 名称 + 特征 + 单位 |
| 人材机 | 报价一套 + 成本一套 | 仅成本侧一套（列名略不同） |

系统首期应以 **名称 + 特征 + 单位** 为主；有编码时作为加分项。

### 2.7 样例数据（理解报价 vs 成本）

| 项目名称 | 综合单价 | 成本单价 | 报价人工 | 成本人工 |
|----------|----------|----------|----------|----------|
| 轻钢龙骨隔墙（100mm） | 146.56 | 130.86 | 78.4 | 70 |

可用于：组价后 **利润率 = (综合单价 - 成本单价) / 成本单价** 及人材机差额分析。

## 三、成本分析表（深海大厦）

### 3.1 工作簿结构

| Sheet | 行数(约) | 有效清单行 |
|-------|----------|------------|
| 装修工程清单 | 451×15 | ~441 |
| 安装工程清单 | 341×15 | ~331 |

### 3.2 表头结构（双行表头）

- **第 3 行（索引 2）**：序号、项目编码、项目名称、项目特征描述、计量单位、工程量、金额(元)  
- **第 4 行（索引 3）**：综合单价、合价、其中：材料设备暂估合价、成本单价、主材、损耗率、人工、辅材、机械  

### 3.3 列映射

| 列号 | 表头 | 系统字段 |
|------|------|----------|
| 0 | 序号 | `seq` |
| 1 | 项目编码 | `list_code` |
| 2 | 项目名称 | `name` |
| 3 | 项目特征描述 | `feature` |
| 4 | 计量单位 | `unit`（需去掉换行，如 `计量\n单位` → `单位`） |
| 5 | 工程量 | `quantity` |
| 6 | 综合单价 | `unit_price` |
| 7 | 合价 | `amount` |
| 8 | 其中：材料设备暂估合价 | `provisional_material` |
| 9 | 成本单价 | `cost_unit_price` |
| 10～14 | 主材、损耗率、人工、辅材、机械 | 成本人材机 |

**有效行**：`项目编码` 或 `项目名称` + `计量单位` 齐全；编码常为 12 位数字。

### 3.4 与九亿表统一策略

入库时统一为内部模型 `BoqLine`：

| 统一字段 | 九亿来源 | 深海来源 |
|----------|----------|----------|
| list_code | 空 | col1 |
| name | col1 | col2 |
| feature | col2 | col3 |
| unit | col3 | col4 |
| unit_price | col5 | col6 |
| cost_unit_price | col15 | col9 |
| labor / material / … | col9-13 / col16-20 | col10-14 |

## 四、对系统导入模块的落地要求（更新）

### 4.1 导入配置模板

建议预置两种 **列映射模板**：

1. `template_guangwan_bid` — 九亿类投标报价清单（23 列，双价人材机）  
2. `template_gb_cost` — 深海类国标+成本分析（15 列，双行表头）

### 4.2 解析参数

| 参数 | 九亿-装修 | 九亿-消防 | 深海 |
|------|-----------|-----------|------|
| header_row | 2 | 2 | 2（+ subheader 3） |
| data_start_row | 3 | 3 | 4 |
| sheet_filter | 装修工程清单, 消防工程 | — | 全部清单 sheet |
| skip_duplicate_price_col | col8 | — | — |

### 4.3 匹配与组价默认值（基于样本）

| 配置项 | 建议默认值 |
|--------|------------|
| 匹配键 | name + feature + unit |
| list_code 权重 | 有编码时 +10% 置信度 |
| 推荐单价字段 | `unit_price`（综合单价不含税） |
| 成本分析字段 | `cost_unit_price` + 主材/人工/辅材/机械 |
| 单位规范化 | ㎡、m²、平方 → `㎡`；`计量\n单位` 清洗 |

### 4.4 导出底稿（在九亿表基础上追加列）

保持原表结构，新增：

- 推荐综合单价、匹配等级、来源工程、历史样本数  
- 推荐成本单价、推荐人工/主材/辅材/机械（来自历史）  
- 审核单价、审核备注
