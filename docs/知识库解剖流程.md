# 知识库解剖流程（每次给资料必走）

> 您每次提供的 Excel 清单+成本，系统都会按本流程**自动解剖**，无需另行叮嘱。

## 一、入库时自动做什么

```text
Excel 资料
  → 识别表头、解析清单行
  → 拆分：主材 / 损耗 / 辅材 / 人工 / 机械 / 成本单价
  → 识别城市、价格档位（高/中/低）
  → 归并「标准项」（名称 + 单位 + 做法签名）
  → 写入 cost_records（原始样本）
  → 聚合 line_price_facts（相似清单项 × 城市 × 档位）
  → 聚合 material_price_facts（材料规格 × 城市 × 档位，如 tile:600*600）
  → 导出「解剖报告」Excel
```

命令：

```powershell
python app.py learn "清单数据资料/AI学习清单/某项目.xlsx" --city 深圳 --tier mid
```

可选参数：

| 参数 | 说明 |
|------|------|
| `--city` | 城市（不填则从文件名/工程名推断） |
| `--tier` | `high` / `mid` / `low`（高/中/低档） |
| `--no-anatomy` | 不生成解剖报告（默认会生成） |

输出：

- SQLite：`data/cost_pricing.db`
- 解剖报告：`data/exports/解剖报告/解剖_工程名_p{id}.xlsx`

## 二、知识库结构

| 层级 | 表 | 用途 |
|------|-----|------|
| 原始样本 | `cost_records` + `standard_items` | 每条历史清单行的完整人材机 |
| 清单项价库 | `line_price_facts` | 相似「名称+做法」在同城同档的中位价 |
| 材料价库 | `material_price_facts` | 同规格主材价（如 600×600 地砖） |
| 材料判定目录 | `material_catalog` | `主材辅材判定.xlsx`：品类主/辅材归类 |
| 定额子目（规划） | `quota_items` | 广联达深圳定额（PDF 待 OCR 录入） |

```powershell
python app.py import-catalog   # 默认导入 主材辅材判定.xlsx
```

## 三、招标组价时怎么用

1. **整项参照**：招标「名称 + 特征」与历史**均相似** → 套主材/辅材/人工/机械（优先同城同档）。  
2. **材料价库**：整项不够像 → 按规格查主材（如瓷砖型号）。  
3. **跨区/跨档**：本地无样本时，按 城市→通用、档位→邻近档 逐级放宽，备注标明。

```powershell
python app.py tender "招标清单.xlsx" --city 深圳 --tier mid --price --export
```

## 四、维护命令

```powershell
python app.py backfill-kb    # 老库回填城市/档位并重建价库
python app.py stats          # 查看项目数、标准项、价库条数
```

配置：`config/regions.json`（城市关键词、档位关键词）、`config/material_rules.json`（材料品类）。
