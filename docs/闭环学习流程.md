# 闭环学习流程（甲方 → 系统 → 校正 → 自我升级）

> **您要的最终结果**：在**不改变甲方 Excel 表结构**的前提下，追加成本拆解列、同表去重公式链接、母行/链接行填色；组价必须按**名称 + 项目特征工序**判断，不能盲目套主材编号整价。

---

## 三份 Excel 各是什么

| 阶段 | 存放位置 | 谁产生 | 作用 |
|------|----------|--------|------|
| **① 甲方招标清单** | `清单数据资料/甲方招标清单/` | 甲方 | 原始工程量与特征，通常无成本或仅有综合单价 |
| **② 系统成本分析** | `data/exports/*_成本分析_去重链接.xlsx` | 本系统 `deliver` | 原表 + 主材/人工/辅材/机械/管理费/利润/税金 + 去重 `$母行` 链接 |
| **③ 金标准（您校正）** | `清单数据资料/AI学习清单/` | **您** | ② 基础上人工改对的最终价，是系统学习的唯一权威 |

命名建议：金标准用 **`项目名-YYYYMMDD.xlsx`**（如 `珠海海德公馆售楼处会所-20260627.xlsx`），与招标日期区分。

## 标准操作（复制即用）

```powershell
# 1. 交付：从甲方清单生成成本分析
python app.py deliver --project haide_sales
# 或指定文件
python app.py deliver "清单数据资料/甲方招标清单/xxx.xlsx" --city 珠海 --tier mid

# 2. 您在 Excel 中改 ②，另存为 ③ 到 AI学习清单/

# 3. 校准：对比 ② vs ③，出升级建议
python app.py calibrate --project haide_sales

# 4. 反哺：把金标准 learn 进知识库 + 重建价库
python app.py calibrate --project haide_sales --learn
python app.py backfill-kb

# 5. 无样本大偏差项：外部搜索参考
python app.py external-price --from-calibration --project haide_sales --limit 5
# 核实 config/external_price_hints.json 后，手工并入 market_reference_prices.json

# 6. 全项目巡检
python app.py calibrate --all
```

项目配对登记在 **`config/project_pairs.json`**（招标路径、金标准路径、导出路径、城市档位）。

## 系统如何「自我升级」

```text
deliver 组价
    ↓
您校正 → AI学习清单（金标准）
    ↓
calibrate 对比偏差聚类
    ↓
┌──────────────────────────────────────────┐
│ learn 金标准 → line_price_facts 整项历史   │
│ backfill-kb → 工艺价型 / 材料价库          │
│ 改 config/market_reference_prices.json    │
│ 改 material_process_price 工序规则         │
│ external-price → external_price_hints     │
│ audit-judge 监控是否接近 80% 核心准确      │
└──────────────────────────────────────────┘
    ↓
下一轮 deliver 更准确
```

**升级优先级**（与 `component_judge` 一致）：

1. 金标准整项匹配（最可信）
2. 项目主材编号 + **工序分解**（PT-01≠整项 110 元/㎡）
3. 涂料 `paint_process` 按特征拆价
4. 市场参考 + 外部搜索 hints
5. 工艺价型（样本≥3 且通过 sanity）

## 交付物检查清单（您验收 ② 时）

- [ ] 原招标列未被破坏（序号/名称/特征/工程量/综合单价公式）
- [ ] 成本区列齐全，母行有数、重复行 `$母行` 链接
- [ ] 母行浅绿 `#C6EFCE`、链接行浅黄 `#FFF2CC`
- [ ] 涂料/PT 编号项：说明里可见工序拆价，非整价套主材表
- [ ] MT 线条：按展开面 mm 计 m，非 450 元/㎡ 整板价
- [ ] 合价总量与您的经验量级接近（校准报告可看 ±10% 覆盖率）

## 新增项目时

1. 甲方 xlsx 放入 `甲方招标清单/`
2. 在 `config/project_pairs.json` 增加一条 `pairs`（`id/tender/gold/export/city/tier`）
3. `python app.py deliver --project <id>`
4. 校正后 `calibrate --learn`
