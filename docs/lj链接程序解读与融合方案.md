## 一、程序是做什么的？

`lj.exe` 解决的是：**几百上千条清单里，大量重复项只需填一次单价，其余行自动“链接”到母表单元格**。

两个核心命令（从反编译字符串还原）：

| 命令 | 用法 | 作用 |
|------|------|------|
| **tj** | `lj.exe tj 清单.xls 参数…` | **去重**：按「名称列 + 单位列」抽出唯一清单项，生成 `tjall-*.xlsx` |
| **lj** | `lj.exe lj 清单.xls 参数…` | **本表链接**：相同项的价格列写 Excel 公式，指向首次出现那一行 |
| **exlj** | `lj.exe exlj 母表.xls 子表.xls 参数…` | **跨表链接**：子表匹配母表，公式引用母表单元格 |
| **extj** | `lj.exe extj …` | 跨表去重变体 |

参数格式（多段，用 `-` 分隔），每段大致为：

```
Sheet序号, 总行数, 名称列1, 名称列2, …, 工程量列, 单位列, 链接单价列1, 链接单价列2, …, -, 总列数
```

---

## 二、匹配规则（lj 原版）

```python
# 等价逻辑
相同项 = (名称列表完全相同) AND (单位相同)
```

**注意：原版不包含「项目特征」**，只靠名称+单位。  
名称可以是**多列**拼成 `itemnames` 列表（支持双名称列等）。

链接方式：在目标单元格写入 Excel 公式，例如：

```excel
='[母表.xls]Sheet1'!$F$128
```

并给已链接区域上色（ColorIndex 43/45）。

---

## 三、去重（tj）流程

1. 打开 Excel，按参数读指定 Sheet  
2. 逐行读：名称（可多列）、工程量、单位  
3. 按 **names + unit** 合并为 `tjitem`  
4. 输出 `tjall-{原文件名}.xlsx`：**去重后的清单**，只需在此表填一次综合单价/成本  
5. 再跑 **lj / exlj**，大表自动引用去重表

**价值**：1000 行可能只有 200 个唯一项 → 人工只填 200 次。

## 四、与我们系统的对比

| 能力 | lj.exe | 本系统（已建/将建） |
|------|--------|---------------------|
| 匹配键 | 名称 + 单位 | **名称 + 单位 + 做法标签 + 特征** |
| 去重 | tj 命令 | **`dedupe` 命令**（更准） |
| 链接 | Excel 公式 | **同工作簿「去重价表」Sheet + 公式 / 内部 ID** |
| 知识库 | 无 | SQLite 历史成本 |
| 表头识别 | 手工参数 | 自动识别 |
| 运行 | Win32 + Excel 必开 | Python 3，openpyxl |

**结论：思路必须吸收**（去重母表 + 链接填价），**匹配应升级**（加入项目特征/做法），**不必依赖老 exe**。

## 五、已融入本项目的功能

| 命令 / 界面 | 说明 |
|------|------|
| `python app.py dedupe 清单.xlsx` | 导出「去重母表」+ 行号映射 |
| `python app.py link-price 招标.xlsx --master 母表.xlsx` | 全量清单公式链接母表 |
| `python app.py tender … --price --export --dedupe-link` | **组价 + 18列导出 + 去重母表链接（推荐大清单）** |
| Streamlit「去重链接」页 | 纯去重 / 母表填价后链接 |
| Streamlit「招标组价」 | 默认勾选去重链接导出 |

导出结构（`--dedupe-link`）：
- **去重母表**：152 项填主材/人工/辅材/机械一次即可
- **单价分析**：J～N 列 `=去重母表!$G$2` 公式链接，黄色=重复项
- **说明**：去重统计（如 355行→152项，少填57%）

## 七、老程序局限

- Python 2.6，32 位，需本机装 Excel  
- 参数手工配置，易 `Input error!`  
- **不识别项目特征** → 三遍漆与两遍漆会误链  
- 无历史库、无成本拆解
