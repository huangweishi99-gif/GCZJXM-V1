# 远程同步：Tailscale + Windows 开机自启 sync serve

> 目标：在外面用手机 **远程指挥家里/公司电脑** 跑 `deliver`，或浏览最新组价。  
> 前提：电脑 **开机、已登录 Windows、不断网、勿休眠**。

---

## 一、整体架构（先看清再装）

```text
  [外面手机]                    [家里/公司电脑]
       │                              │
       │  Tailscale 虚拟专网 (100.x.x.x)
       └──────────────┬───────────────┘
                      │
              http://100.x.x.x:8765/
                      │
              python app.py sync serve
                      │
              本地 Excel + cost_pricing.db
```

| 组件 | 作用 |
|------|------|
| **Tailscale** | 外网也能像局域网一样访问电脑，无需公网 IP |
| **sync serve** | 本项目的同步 API + 手机网页 |
| **API Token** | 防止陌生人远程跑 deliver |

## 二、Tailscale 安装检查清单

### A. 公司/家里电脑（Windows）

#### 步骤 1：下载安装

1. 浏览器打开：**https://tailscale.com/download**
2. 选 **Windows** → 下载并运行安装包
3. 安装向导一路 **Next** → **Install** → 完成

**界面应类似：**

```text
┌─────────────────────────────────────┐
│  Tailscale Setup                    │
│  [ Install ]  [ Cancel ]            │
└─────────────────────────────────────┘
```

#### 步骤 2：登录同一账号

1. 安装完成后托盘出现 Tailscale 图标（右下角 ^ 展开）
2. 点击 **Log in**
3. 用 **Google / Microsoft / GitHub** 等登录（记住这个账号）

**检查点：**

```text
┌─────────────────────────────────────┐
│  Tailscale                          │
│  ● Connected                        │
│  本机名称: DESKTOP-XXXX             │
│  Tailscale IP: 100.xx.xx.xx   ← 记下 │
└─────────────────────────────────────┘
```

#### 步骤 3：确认 IP 地址

- 右键托盘 Tailscale → **Preferences** 或打开主窗口
- 找到 **This device** 下的 **100.x.x.x**（每台机器不同）
- 手机远程访问时用：`http://100.x.x.x:8765/`

#### 步骤 4：Windows 防火墙（首次访问若失败再做）

1. **设置** → **隐私和安全性** → **Windows 安全中心** → **防火墙和网络保护**
2. 若手机连不上 8765，可临时测试：允许 **Python** 专用网络
3. 更稳妥：仅 Tailscale 网卡放行（高级用户）；多数家庭网络允许 Python 即可

**检查命令（PowerShell）：**

```powershell
# Tailscale 是否在线
tailscale status

# 本机 Tailscale IP
tailscale ip -4
```

若提示找不到命令，用托盘界面查看 IP 即可。

#### 步骤 1：安装 App

- **iOS**：App Store 搜 **Tailscale**
- **Android**：应用商店或 https://tailscale.com/download

#### 步骤 2：登录 **与电脑相同的账号**

**检查点：**

```text
┌─────────────────────────────────────┐
```

- 若看不到：检查电脑 Tailscale 是否 Connected、是否同一登录账号

## 三、本项目：Token + 首次手动启动

在 **项目目录** 打开 PowerShell：

```powershell
cd "C:\Users\guangwan\Documents\工程造价项目-V1"
pip install -r requirements.txt
python app.py sync token          # 生成 Token，抄到手机「设置」页
python app.py sync serve          # 先手动跑一次，确认无报错
```

**手机浏览器测试：**

1. 手机开 Tailscale
2. 访问 `http://100.x.x.x:8765/`（换成你的 IP）
3. **设置** → 填地址 + Token → **远程** → 试「跑 deliver」

> **安全**：`config/sync_server.json` 含 Token，**不要** `git push` 到 GitHub。

## 四、Windows 禁止休眠（远程必备）

否则 sync serve 在睡眠后手机连不上。

1. **设置** → **系统** → **电源和电池**（或「电源和睡眠」）
2. **屏幕关闭**：按需（可 10 分钟）
3. **睡眠**：**从不**（接通电源时）
4. 笔记本：**接通电源** 与 **使用电池** 都建议「睡眠 → 从不」若需长期远程

**界面应类似：**

```text
┌─────────────────────────────────────┐
│  接通电源时                          │
│  关闭屏幕：10 分钟                     │
│  使计算机进入睡眠状态：从不  ← 重要    │
└─────────────────────────────────────┘
```

## 五、Windows 开机自启 sync serve（推荐：任务计划程序）

比「启动文件夹」更稳：可设 **工作目录**、失败重试、写日志。

### 步骤 1：确认启动脚本存在

项目内脚本：

`scripts\windows\sync_serve_at_logon.bat`

若 `python` 不在 PATH，用记事本打开该 bat，把 `set PY=python` 改成 Python 完整路径。

### 步骤 2：打开任务计划程序

1. `Win + R` → 输入 **`taskschd.msc`** → 回车
2. 右侧 **创建任务…**（注意是「创建任务」，不是「创建基本任务」）

### 步骤 3：「常规」选项卡

```text
┌─────────────────────────────────────┐
│  名称: 工程造价-sync-serve           │
│  ☑ 不管用户是否登录都要运行          │  ← 可不勾，勾了需输密码
│  ☑ 使用最高权限运行                  │  ← 一般可不勾
│  配置: Windows 10 / 11               │
└─────────────────────────────────────┘
```

**建议（简单版）：**

- **不勾**「不管用户是否登录都要运行」
- **不勾**「使用最高权限」
- 即：**用户登录后自动启动**（足够家用/办公室）

### 步骤 4：「触发器」选项卡

1. **新建…**
2. 开始任务：**登录时**
3. 特定用户：你的 Windows 用户名
4. **启用** → 确定

```text
┌─────────────────────────────────────┐
│  触发器: 登录时                      │
│  用户: DESKTOP\你的用户名            │
└─────────────────────────────────────┘
```

### 步骤 5：「操作」选项卡

1. **新建…**
2. 操作：**启动程序**
3. 程序或脚本（浏览选择）：

   `C:\Users\guangwan\Documents\工程造价项目-V1\scripts\windows\sync_serve_at_logon.bat`

4. **起始于（重要）**：

   `C:\Users\guangwan\Documents\工程造价项目-V1`

5. 确定

```text
┌─────────────────────────────────────┐
│  程序: ...\sync_serve_at_logon.bat   │
│  起始于: ...\工程造价项目-V1         │  ← 必须填项目根目录
└─────────────────────────────────────┘
```

### 步骤 6：「条件」选项卡

- **取消勾选**：「只有在计算机使用交流电源时才启动」  
  （若笔记本只想插电时跑，可保留勾选）

### 步骤 7：「设置」选项卡

- 勾选：**如果任务失败，按以下方式重新启动** → 间隔 **1 分钟**，尝试 **3 次**

### 步骤 8：保存并测试

1. 确定保存任务（可能要求输入 Windows 密码）
2. 在任务列表找到 **工程造价-sync-serve** → 右键 **运行**
3. 等 10 秒，手机或本机浏览器访问：

   `http://127.0.0.1:8765/api/health`

   应返回 JSON：`"ok": true`

4. 查看日志（若有报错）：

   `data\sync\serve.log`

### 步骤 9：重启电脑验证

1. 重启 → 登录 Windows
2. 等 30 秒
3. 本机浏览器打开 `http://127.0.0.1:8765/api/health`
4. 手机 Tailscale 打开 `http://100.x.x.x:8765/`

## 六、Tailscale 是否也要开机自启？

- Tailscale Windows 客户端 **默认随系统启动**
- 托盘图标为 Connected 即可
- 若未自启：**设置** → **应用** → **启动** → 打开 **Tailscale**

## 七、常见问题

| 现象 | 处理 |
|------|------|
| 手机看不到电脑 | 两设备是否同一 Tailscale 账号；电脑是否 Connected |
| 能 ping 100.x 但网页打不开 | 电脑上 `sync serve` 是否在跑；查 `data\sync\serve.log` |
| 401 无效 Token | 手机设置页 Token 与 `python app.py sync token --show` 一致 |
| 电脑睡眠后断连 | 睡眠设为「从不」；合上笔记本可改合盖「不操作」 |
| deliver 很慢 | 正常，大项目 1～3 分钟；远程页会轮询任务状态 |
| GitHub/Cursor 手机无最新价 | 组价数据在本地，不靠 GitHub；必须用 sync 网页或 Tailscale |

## 八、每日使用速查

**电脑端（长期开着即可）：**

- Tailscale：Connected

2. 浏览器 `http://100.x.x.x:8765/`
3. **远程** → 选项目 → **跑 deliver**

**电脑上拉回手机校正：**

```powershell

```
