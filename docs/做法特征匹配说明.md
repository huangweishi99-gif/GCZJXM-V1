# 做法 / 项目特征匹配说明

## 为什么必须拆做法

同名清单（如「内墙乳胶漆」「地砖地面」）单价可以差很多，差别在**特征描述里的做法**：

| 对比 | 识别标签 | 库内分条 |
|------|----------|----------|
| 乳胶漆三遍 vs 两遍 | `paint_coats:3` / `2` | 不同 `method_signature` |
| 水泥砂浆贴砖 vs 瓷砖胶 | `tile_bind:mortar` / `adhesive` | 不同标准项 |

## 匹配权重（默认）

- 做法标签 **45%**
- 特征全文 **35%**
- 项目名称 **20%**

## 冲突处理

关键标签不一致时：

- 一致度封顶，等级降为 C
- **不自动填价**，备注写明「做法冲突[…]」

## 扩展规则

编辑 `config/feature_rules.json` 增加 `pattern` → `key` / `value` / `label`。

入库或回填：

```bash
python app.py learn "新项目.xlsx"
python app.py backfill   # 老库补标签
```
