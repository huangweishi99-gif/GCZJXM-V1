# 甲方 vs 投标方表头对照（2026 更新版）

> 样本：`清单数据资料/AI学习清单/甲方一般表头S投标方需求一般表头.xls`

---

## 一、需求确认（已与用户对齐）

| # | 约定 |
|---|------|
| 1 | 管理费 / 利润 / 税金 **暂固定 0%**（P4、Q4、R4 = 0） |
| 2 | 甲方表**列数不固定**，以智能识别为准；常见约 8 列，**备注可有可无** |
| 3 | **需报价行**：同时有 **项目名称 + 单位 + 工程量** |
| 4 | 表格样式多变 → **自动识别表头**，不依赖固定列号 |
| 5 | `learn` 自主学习您提供的资料入库；`tender`+`price`+`export` 生成投标方单价分析表 |

## 二、更新后投标方表（18 列 A～R）

| 列 | 行4子表头 | 说明 |
|----|-----------|------|
| A～E | 序号、名称、特征、单位、工程量 | 与甲方相同（招标带入） |
| F～H | 综合单价、合价、备注 | 上报甲方；备注可选 |
| I～N | 成本单价、主材、损耗率、人工、辅材、机械 | 成本单价分析区 |
| **O** | **合价** | **成本合价** = 成本单价 × 工程量 |
| P～R | 管理费、利润、税金 | 费率行固定 **0** |

### 公式链（第 6 行样例，与 Excel 一致）

```
I = J + J×K + L + M + N          成本单价
O = I × E                        成本合价
P = I × $P$4                     管理费（当前 0）
Q = (I + P) × $Q$4               利润（当前 0）
R = (I + P + Q) × $R$4           税金（当前 0）
F = I + P + Q + R                综合单价（0 费率时 F = I）
G = E × F                        投标合价
```

甲方表仅：`G = E × F`（招标阶段 F 常为空）。

## 三、与上一版差异

| 项目 | 旧版 | 新版 |
|------|------|------|
| 总列数 | 17 (A～Q) | **18 (A～R)** |
| 成本区「合价」 | 无独立列 | **O 列**（=I×E） |
| 管理/利润/税金 | O/P/Q 列 | **P/Q/R 列** |
| 费率 | 3.5% / 5% / 9% | **0 / 0 / 0** |

---

## 四、系统识别规则

1. 扫描前 35 行，按别名匹配表头（支持双行表头、合并单元格文字）。  
2. 多个「合价」：靠前且靠近综合单价 → 投标合价；靠后且在成本区 → 成本合价。  
3. 两套「主材」列 → 第二套视为成本侧。  
4. 跳过分部行（有名称、无单位或无工程量）。  
5. 跳过 Sheet：封面、汇总、说明、品牌表等。

## 五、命令速查

```powershell
python app.py learn "AI学习清单\xxx.xlsx"    # 新资料 → 知识库
python app.py tender "招标清单.xlsx" --price --export
```
